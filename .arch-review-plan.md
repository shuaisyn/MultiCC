## P1 — 次做（中价值，多为 DRY/死码）

- **providers.js 死码**：`applyWireDefaults`（已内联进 resolveSpawnEnv）、`isAliasOnlyClaude`、`getProviderRow` 导出、`available()` 导出 → 删除。低风险。
- **前端 DRY（最大frontend收益，需谨慎）**：`CLAUDE_MODEL_OPTIONS`、`providerAliasMap`、`providerAliasTiers`、`normalizeModelForProvider`、`modelShortName`、`modelDisplayName` 在 chat.js 与 manage.js 字节级重复 → 抽 `public/shared/models.js`，两处 `<script>` 引入。风险中（双前端），工作量 M。
- **chat.js showModelPicker 死码**：未被调用的近重复函数 → 删除。低风险。
- **manage.js rebuildModelOptions** 自行内联 tier 构建而非复用 `providerAliasTiers` → 改调用。低风险。
- **tier 词汇魔数**：`['opus','sonnet','haiku','fable']` 硬编码 6-8 处 → 由 `ALIAS_TIER_KEYS` 派生/共享常量。低风险。
- **CLAUDE_ROUTING_KEYS**：server.js 内联重写一份（注释要求手动同步）→ 从 providers.js 导入。低风险。
- **ws-broadcast-loop**（server.js）：遍历 WS client 发送逻辑重复 8+ 处 → 抽 helper。低风险，工作量 M。
